# Story 1.3: Implement Debug View

## Status: Draft

## Story

- **As a** developer
- **I want** a simple, scrollable text view on the main screen
- **so that** I can log and observe internal states and messages during development.

## Acceptance Criteria (ACs)

1. A non-interactive, scrollable text area is present at the bottom of the main screen.
2. A globally accessible logging utility/service is created.
3. Calling the logging utility appends new messages to the text area.
4. The view automatically scrolls to show the latest message.
5. The entire debug view and its associated logic can be completely removed from production builds using environment-based compile-time flags (e.g., `kDebugMode`).

## Tasks / Subtasks

- [ ] **Task 1: Create the Debug View UI Widget** (AC: 1)
    - [ ] Create a new widget, `DebugView`.
    - [ ] The widget should contain a `Scrollable` (like `ListView` or `SingleChildScrollView`) inside a `Container` with a dark background.
    - [ ] It will display a list of strings.
- [ ] **Task 2: Implement the Logging Service** (AC: 2)
    - [ ] Create a `DebugLogService` as a singleton or provided via `get_it`.
    - [ ] The service should manage a `List<String>` or a `Stream<String>` for log messages.
    - [ ] Create a public method, `log(String message)`, to add messages.
- [ ] **Task 3: Connect Service to UI** (AC: 3, 4)
    - [ ] The `DebugView` widget should listen to the `DebugLogService`.
    - [ ] When a new log is added, the widget rebuilds to show the new message.
    - [ ] Implement auto-scrolling to the bottom of the list when a new message is added. A `ScrollController` will be needed.
- [ ] **Task 4: Implement Compile-Time Flag** (AC: 5)
    - [ ] In `MainScreen`, wrap the `DebugView` widget in a conditional check: `if (kDebugMode)`.
    - [ ] Ensure all calls to the `DebugLogService` are also wrapped in `if (kDebugMode)` checks to avoid executing unnecessary code in production.
    - [ ] Verify that the debug view is not present when running the app in profile or release mode.

## Dev Technical Guidance

- **`kDebugMode`**: This is a compile-time constant provided by `package:flutter/foundation.dart`. Use it to conditionally include or exclude code. This is more efficient than a runtime check.
- **Service Locator**: The `DebugLogService` should be registered with `get_it` for easy access from anywhere in the app.
- **State Management for Logs**: To make the UI reactive, the `DebugLogService` could use a `ValueNotifier` or a BLoC/Cubit to expose the list of logs. A simple `StreamController` is also a very effective option.
- **Performance**: While this is a debug tool, avoid performance issues. If logs become very numerous, consider capping the list size (e.g., only keep the last 100 messages).
