# Story 1.1: Project Initialization & BLoC Structure

## Status: Draft

## Story

- **As a** developer
- **I want** to set up a new Flutter project with the required BLoC architecture and dependencies
- **so that** we have a clean, structured foundation for development.

## Acceptance Criteria (ACs)

1. A new Flutter project is created and configured.
2. The folder structure for BLoC (Business Logic Component) is established according to project guidelines.
3. Required dependencies (`flutter_bloc`, `web_socket_channel`, `camera`, `get_it`, `injectable`) are added to `pubspec.yaml`.
4. The main screen is created with a top area for the camera view and a bottom area for a debug view.
5. The application runs successfully on a target device/emulator, showing the initial placeholder layout without errors.

## Tasks / Subtasks

- [ ] **Task 1: Initialize Flutter Project** (AC: 1)
    - [ ] Create a new Flutter project named `face_check_in_flutter`.
    - [ ] Clean up the default `main.dart` and project structure.
- [ ] **Task 2: Add Dependencies** (AC: 3)
    - [ ] Add `flutter_bloc` for state management.
    - [ ] Add `web_socket_channel` for backend communication.
    - [ ] Add `camera` for video feed.
    - [ ] Add `get_it` and `injectable` for service location and dependency injection.
    - [ ] Run `flutter pub get` to install dependencies.
- [ ] **Task 3: Establish BLoC Architecture** (AC: 2)
    - [ ] Create directories for `lib/application`, `lib/domain`, `lib/presentation`, `lib/infrastructure`.
    - [ ] Set up the basic folder structure for features within the presentation layer.
- [ ] **Task 4: Implement Main Screen Layout** (AC: 4)
    - [ ] Create a new `MainScreen` widget.
    - [ ] Implement a `Scaffold` with two main sections: a larger top area (for camera) and a smaller bottom area (for debug logs).
    - [ ] Use placeholder widgets (e.g., `Container` with colors) for these areas.
- [ ] **Task 5: Verify Initial Build** (AC: 5)
    - [ ] Configure `main.dart` to launch the `MainScreen`.

    - [ ] Confirm the main layout appears as expected.

## Dev Technical Guidance

- **Project Naming:** Ensure the project is created with the name `face_check_in_flutter`.
- **Architecture:** Follow the Clean Architecture principles as outlined in the main `architecture.md`. The BLoC pattern will reside within the `application` and `presentation` layers.
- **Dependency Injection:** Use `get_it` and `injectable` to set up the dependency injection framework early. This is crucial for decoupling services like the `WebSocket` and `Camera` handlers later on.
- **State Management:** All UI state changes must be handled via BLoCs. Do not use `StatefulWidget` for managing complex state.
- **Dependencies from PRD:** The PRD explicitly calls for `flutter_bloc` and `web_socket_channel`. The `camera` dependency is implied by the core functionality.
