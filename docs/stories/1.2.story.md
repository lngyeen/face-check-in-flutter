# Story 1.2: Implement Live Camera Preview & Auto-Start

## Status: Draft

## Story

- **As a** user
- **I want** the application to show a live video feed from the camera immediately after granting permission
- **so that** the check-in process can begin automatically.

## Acceptance Criteria (ACs)

1. The application correctly requests camera permissions on its first launch.
2. If permission is granted, the live camera preview is displayed in the designated UI area.
3. The application state transitions to "streaming" upon successful camera initialization.
4. If permission is denied, a user-friendly message is displayed, instructing the user to grant camera permissions from the device settings.
5. The camera preview is stable and does not crash or show errors.

## Tasks / Subtasks

- [ ] **Task 1: Implement Permission Handling** (AC: 1, 4)
    - [ ] Add the `permission_handler` dependency.
    - [ ] Create a Camera BLoC/Cubit to manage camera states (e.g., `Initial`, `PermissionGranted`, `PermissionDenied`, `Streaming`).
    - [ ] On app start, trigger an event to request camera permission.
    - [ ] Handle both "granted" and "denied" responses and update the BLoC state accordingly.
- [ ] **Task 2: Initialize Camera Controller** (AC: 2, 3)
    - [ ] Create a `CameraService` to encapsulate all `camera` plugin logic.
    - [ ] When permission is granted, initialize the `CameraController`.
    - [ ] Select the front-facing camera by default.
    - [ ] Start the camera preview.
- [ ] **Task 3: Display Camera Preview in UI** (AC: 2, 5)
    - [ ] In the `MainScreen` widget, listen to the Camera BLoC state.
    - [ ] When the state is `PermissionGranted` or `Streaming`, display the `CameraPreview` widget from the initialized controller.
    - [ ] Ensure the preview widget fits correctly within its layout container.
- [ ] **Task 4: Display Permission Denied Message** (AC: 4)
    - [ ] In the `MainScreen` widget, when the Camera BLoC state is `PermissionDenied`, display a clear message to the user.
    - [ ] The message should be helpful, explaining why the permission is needed and how to grant it.

## Dev Technical Guidance

- **Camera Plugin**: Use the official `camera` plugin for Flutter.
- **State Management**: The camera's lifecycle (permissions, initialization, streaming state) MUST be managed by a dedicated BLoC or Cubit. This decouples the UI from the camera logic.
- **Error Handling**: The `CameraController` can throw exceptions. Ensure all initialization and control logic is wrapped in `try-catch` blocks and that errors are reported to the BLoC.
- **Asynchronous Operations**: All interactions with the camera and permissions are asynchronous. Use `async/await` correctly and provide loading indicators in the UI if initialization takes time.
